# 项目规范（Project Guidelines）

## 1. 项目介绍

本项目为个人博客网站，采用以下技术栈构建：

- **Next.js**：应用框架
- **Zustand**：状态管理
- **Tailwind CSS**：样式系统
- **Motion**：动画库

项目目标：

- 保持代码结构清晰
- 保持风格统一
- 遵循现代前端最佳实践
- 提高可维护性与可扩展性

---

## 2. 开发约束

### 2.1 组件使用规范

1. 新增 UI 组件时，**必须优先使用 shadcn/ui 组件库**。
2. 组件必须通过官方 CLI 命令添加：

   ```bash
   npx shadcn add <组件名称>
   ```

3. 严禁手动复制 shadcn 组件源码。
4. 严禁绕过 CLI 直接创建 shadcn 组件结构。
5. 如确有必要自定义组件，必须：
   - 保持与现有组件体系风格一致
   - 遵循项目命名规范
   - 不破坏现有组件结构

---

### 2.2 contents 目录的文件规范

1. `contents` 目录为博客内容产出目录。
2. 新增内容时必须遵循当前既有命名规范。
3. 创建新内容文件时：
   - 仅需提供简要占位内容
   - 不需要生成大篇幅正文
   - 不要编造完整博客文章内容

---

## 3. 代码输出要求

所有新增或生成的代码必须满足以下要求：

### 3.1 规范要求

- 严格符合项目现有：
  - ESLint 配置
  - Prettier 格式化规则
- 保持代码风格一致

### 3.2 提交前必须满足

- 无 ESLint 报错
- 无格式冲突
- 不引入未使用变量
- 不产生类型错误（如使用 TypeScript）
- 不生成无关代码

---

## 4. 其他约定

1. 保持目录结构清晰，不随意新增顶层目录。
2. 优先复用已有组件与工具函数。
3. 避免重复实现已有功能。
4. 所有样式必须优先使用 Tailwind CSS。
5. 禁止随意编写全局 CSS。
6. 禁止引入未使用依赖。
