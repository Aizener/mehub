# 版本号生成命令

按照语义化版本规范（x.y.z）更新 `package.json` 中的版本号，并生成对应的 Git 提交信息。

---

## 功能适用范围

- 当且仅当存在**功能代码变更**时才允许修改 `package.json` 中的版本号；
- 如果只是 `contents/*` 里的博客文章、日常碎碎念的新增、修改或删除，不影响网站功能的，不进行版本号更新；
- 例如对 `.cursor` 目录的修改，不进行操作；
- 文档、样式、注释、配置文件等不影响功能行为的变更，不进行版本号更新。

> 核心原则：仅当网站“功能行为发生变化”时才更新版本号。

---

## 功能说明

本命令将：

1. 修改 `package.json` 中的 `version` 字段；
2. 支持以下输入方式：
   - 指定版本号（如 1.2.3）
   - major
   - minor
   - patch
3. 校验版本号格式是否合法（x.y.z）；
4. 生成符合 Conventional Commits 规范的 Git message；
5. 可选执行 `git commit` 与 `git tag`。

---

## 执行步骤

### 第一步：读取当前版本

从 `package.json` 中读取：

```json
{
  "version": "x.y.z"
}
```

---

### 第二步：计算新版本号

- 输入 `major` → 主版本号 +1，次版本号和修订号重置为 0；
- 输入 `minor` → 次版本号 +1，修订号重置为 0；
- 输入 `patch` → 修订号 +1；
- 输入明确版本号 → 直接使用输入值（必须符合 x.y.z 格式）。

版本格式必须符合：

```
^\d+\.\d+\.\d+$
```

---

### 第三步：修改 package.json

仅替换：

```json
"version": "旧版本号"
```

为：

```json
"version": "新版本号"
```

不得修改其他字段内容。

---

### 第四步：生成 Git 提交信息

提交信息格式为：

```
chore(release): 版本号更新至 新版本号
```

示例：

```
chore(release): 版本号更新至 新版本号
```

或：

```
chore(package): 版本号更新至 0.3.6
```

如果涉及`major`或者`minor`此类版本号的更改，需要把`tag`的提交信息一起输出。
